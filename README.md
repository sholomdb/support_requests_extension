# מזין בקשות סיוע – Chrome Extension

תוסף Chrome שמסייע בהזנת בקשות סיוע מאקסל לאתר, עם מעקב יתרות לפי עיר.

## מה התוסף עושה

1. **טוען קובץ אקסל** – מזהה שורות נתונים (שורה 6 = כותרות, שורה 7+ = נתונים)
2. **מעבד את כל השורות מיד עם הטעינה** – תיקון ואימות (ראו "איך עובד העיבוד" למטה),
   כולל כל שאלות המיפוי לאופרטור, כך שאין יותר עיבוד או שאלות תוך כדי מילוי
3. **מציג את השורות** – שם, ת.ז., טלפון, תאריך לידה, כתובת, סוג בקשה, סכום, וסטטוס
4. **ממלא טפסים באתר** – שלב אחר שלב (פרטים → אוכלוסייה → בקשה → סכום), מתוך
   הנתונים שכבר עובדו – ללא המתנה לתשובות מיפוי
5. **מאמת יתרות** – משווה סכום השורות להפרש היתרות, ושומר יתרה לפעם הבאה
6. **מייצא טבלה + יומן** – קובץ .xlsx עם הטבלה המעובדת והיומן, לצפייה מחוץ לתוסף

## איך עובד העיבוד (pipeline)

מיד עם טעינת הקובץ – עוד לפני שממלאים שורה אחת – התוסף:

1. קורא את העמודות הגולמיות מהאקסל
2. סורק את **כל** השורות ואוסף כל ערך קטגוריאלי לא ידוע (עיר, סוג תקציב, פריט,
   ארץ לידה, סיווג משפחה, מקור תקציב) – ושואל על **כל ערך ייחודי פעם אחת בלבד**,
   לא על כל שורה בנפרד
3. מתקן (padding, נרמול תאריכים/טלפון, מיפוי) ומאמת (שדות חובה, פורמט ת.ז., סכום)
   כל שורה, ומייצר טבלה של "בקשות" מוכנות למילוי – עם סטטוס (✓ מוכן / ⚠ לא תקין /
   ? דורש מיפוי) ומעקב לפי שלב (1/2/3) לכל שורה

**לתיקון שורה** – פשוט לתקן את התא באקסל ולטעון את הקובץ מחדש. התוסף מזהה שורות
לפי הקובץ + ת.ז. + מספר שורה, כך שההתקדמות שכבר בוצעה (שלבים שכבר מולאו) נשמרת,
ורק השורה שהשתנתה מתעדכנת. אין צורך לסנן שורות "גמורות" – רשימת השורות בפאנל
הצד תמיד מציגה את כל השורות עם הסטטוס שלהן.

מיפויים שנשמרו ניתנים לעריכה/מחיקה בכל עת בעמוד ההגדרות, או דרך כפתור
"תקן מיפויים" בפאנל הצד (מריץ מחדש את שלב המיפוי בלי לטעון קובץ מחדש).

לפירוט מלא של כל שדה (מקור, תיקון, אימות) ראו [`docs/PIPELINE.md`](docs/PIPELINE.md).

## התקנה

1. פתח Chrome → `chrome://extensions`
2. הפעל **Developer mode**
3. לחץ **Load unpacked**
4. בחר את התיקייה `extension/`

## הגדרה ראשונית

1. לחץ על ⚙️ בפאנל הצד → **הגדרות**
2. הזן **ת.ז. התחברות** לכל עיר (ביתר עילית, אלעד וכו')
3. ניתן לגבות/לשחזר את הערים וההגדרות הכלליות דרך **ייבוא/ייצוא הגדרות** (JSON)

**Selectors** לשדות באתר מוגדרים בקוד (`extension/shared/config.js`, `DEFAULT_SELECTORS`)
ולא דרך ההגדרות – לשינוי, ערכו את הקובץ ישירות. חריג: selector של פריט בקטלוג
נלכד דרך "לכוד סלקטור" בטבלת המיפויים בהגדרות (ראו `docs/PIPELINE.md`).

### שלבי הטופס באתר

| שלב | שדות מהאקסל |
|-----|-------------|
| 1 – פרטים | ת.ז., שם, מגדר, מגזר, מצב משפחתי, תאריך לידה, טלפון, כתובת |
| 2 – אוכלוסייה | סוג תקציב (עמודה A) |
| 3 – בקשה | פריט (R), סעיף פריט (S) |
| 4 – סכום | סכום (T), נימוק (U) |

## שימוש

1. התחבר לאתר עם ת.ז. של העיר (SMS – ידנית)
2. פתח את פאנל הצד (לחץ על אייקון התוסף)
3. **טען קובץ אקסל**
4. מילוי אוטומטי:
   - **▶ מלא בקשה** – עובר לדף הבית, יוצר "רשומה חדשה", וממלא את כל השלבים של
     הבקשה הנוכחית. עוצר בשלב הראשון שנכשל כדי שתשלים אותו ידנית.
   - **⏩ מלא בקשות מהנוכחית** – מריץ "מלא בקשה" לכל הבקשות מהנוכחית והלאה. בקשה
     שנכשלת מסומנת בשגיאה, והתוסף חוזר לדף הבית וממשיך לבקשה הבאה. לחיצה נוספת
     על הכפתור (שהופך ל"■ עצור") עוצרת בסוף הבקשה הנוכחית.
5. מילוי ידני (שלב-שלב) – כפתורי **מלא 1. MUTAV / מלא 2. CATALOG / מלא 3. WhoHowM**:
   - כל כפתור ממלא את שדות השלב, לוחץ על "הבא"/"שלח" (או בוחר את הפריט ב-CATALOG,
     שם אין כפתור), ומאמת שהדף באמת עבר לדף הבא. השלב נחשב "עבר" רק אם כל השדות
     מולאו כראוי **וגם** הדף התקדם (MUTAV→CATALOG→WhoHowM→דף הבית)
   - שלב שנכשל מסומן ✗ (אדום); שורה נחשבת "ירוקה"/הושלמה רק כששלושת השלבים עברו
   - **✓ סמן כהצלחה / ✗ סמן ככישלון** – מעקף ידני לקביעת סטטוס השורה
   - **מסנן הרשימה** (הכל / לא הושלמו / נכשלו / דורשות תיקון) משפיע גם על הרשימה
     וגם על "מלא בקשות מהנוכחית"

## מבנה קובץ האקסל

התוסף תואם לקבצים כמו `beytar.xlsx` ו-`elad.xlsx`:

- שורה 6: כותרות
- שורה 7+: נתונים (שורות ללא ת.ז. מדולגות)
- עמודות עיקריות: A=סוג תקציב, B=עיר, C=ת.ז., E/F=שם, G–I=מגדר/מגזר/מצב משפחתי, L=תאריך לידה, N=טלפון, O–Q=כתובת, R/S=פריט, T=סכום, U=נימוק

## הערות

- **SMS** – הקוד מוזן ידנית; התוסף לא מתחבר במקומך
- **Selectors** – מוגדרים בקוד (`config.js`), לא בהגדרות
- **תוספות תקציב** – אם היתרה הקודמת שונה מהשמורה, התוסף מציג אזהרה (ייתכן תוספת תקציב)

## קבצים

```
extension/
├── manifest.json
├── popup/          # ממשק ראשי (פאנל צד)
├── content/        # מילוי טפסים בדף (DOM-fill adapter)
├── background/     # service worker
├── options/        # הגדרות ערים, selectors, ולכידת selector לפריטים
├── shared/
│   ├── excel-parser.js  # חילוץ גולמי מהאקסל
│   ├── pipeline.js       # תיקון + אימות + בניית טבלת בקשות (הליבה)
│   ├── mappings.js       # מיפויים קטגוריאליים שנשמרו ע"י האופרטור
│   ├── inference.js      # ניחוש שדות (ארץ לידה, סיווג משפחה)
│   ├── storage.js        # session (כולל requests, log) ב-chrome.storage
│   └── config.js         # הגדרות אתר/ערים/selectors
└── lib/            # SheetJS (xlsx) – משמש גם לייצוא

docs/
└── PIPELINE.md     # טבלת השדות המלאה (מקור → תיקון → אימות → שלב)
```
